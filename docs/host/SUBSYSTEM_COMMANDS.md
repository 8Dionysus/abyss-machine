# Subsystem Command Ownership

`abyss-machine` still exposes one CLI entrypoint, but subsystem behavior should
not be owned by argument parsing code. The command boundary is:

- `src/abyss_machine/cli.py`: argparse wiring, command dispatch, host adapter
  calls, latest/history writes, and human/JSON output.
- package modules under `src/abyss_machine`: stable contracts, policy rules,
  read-model shapes, and pure selection/validation logic.
- `scripts/validators`: public-safe projection and portability checks, not
  hidden product behavior.

## Stop Lines

- Do not move private `/var/lib` or `/srv` evidence into Git.
- Do not make `abyss-machine` own `abyss-stack`.
- Do not remove typing/nervous machinery to simplify publication.
- Do not treat artifact/provenance work as the command-glue refactor.
- Do not turn validators into the source of subsystem behavior.

## Future Update Rule

When a command grows stable reusable logic, move that logic into a package
module first, keep live host adapters at the CLI edge, and add module tests plus
CLI/projection parity tests. Only then update this map and the README status.
