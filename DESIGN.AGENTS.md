# Agent Design Notes

Use this file when changing repo form, route law, or agent-facing surfaces.

## Design Pressure

The repo is public, but the organ it installs is intimate with a machine. The
healthy design is not a clone of the current workstation; it is a reproducible
seed that can render local roots, then let validators and opt-in units create
private evidence on the target host.

## Agent Rules

- Prefer route cards over broad prose when adding a new district.
- Add mechanics only when a recurring host move has clear inputs and outputs.
- Keep privacy gates visible near typing, nervous, browser, transcript, and
  vault-adjacent surfaces.
- Use public examples and schemas to describe shape, not live records.
- When a path can be rendered, keep the source path symbolic and the install
  path explicit in docs.

## Review Questions

- Does this change publish source, or accidentally publish state?
- Does it make a fresh machine more reproducible?
- Is `abyss-stack` consumed read-only unless explicitly routed otherwise?
- Are typing and nervous flows preserved as organs, not trimmed as risk?
